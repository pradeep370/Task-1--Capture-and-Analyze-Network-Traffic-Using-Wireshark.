Task 5 — Wireshark Network Traffic Capture and Analysis

Date: 11 August 2025  |  Submitted by: PRADEEP S

Objective

To capture live network traffic using Wireshark, analyze the captured packets to identify common protocols such as DNS, HTTP, TCP, and ICMP, and understand their roles in network communication.

Tools Used

  • Wireshark — For capturing and analyzing network packets.
  • Command Prompt (ping command) — To generate ICMP traffic for analysis.

Steps Performed

  1. Installed and launched Wireshark.
  2. Selected the active network interface (Wi‑Fi).
  3. Started packet capture in Wireshark.
  4. Ran ping google.com in Command Prompt to generate ICMP traffic.
  5. Visited a website in a browser to create HTTP/DNS traffic.
  6. Stopped the capture after about one minute.
  7. Applied protocol filters in Wireshark (e.g., dns, tcp, http, icmp).
  8. Saved the capture file as task5_capture.pcapng.

Traffic Flow (Simple)

  1. DNS Lookup: The computer asks a DNS server to find the IP address of websites (e.g., google.com).
  2. TCP Connection: The computer and server perform a TCP handshake (SYN, SYN-ACK, ACK) to establish a connection.
  3. Ping Test (ICMP): ICMP echo requests and replies are used to check reachability.
  4. Other Activity: Background UDP packets and DNS requests from apps or OS services.
  5. HTTP/HTTPS Connection: After the TCP handshake, an HTTP or HTTPS request is sent and the server responds with data (HTTPS uses TLS encryption).

0.090000  Your_IP → 142.250.x.x   TCP ACK
0.100000  Your_IP → 142.250.x.x   HTTP GET /

Note: timestamps and IPs are representative samples from the capture.

Protocols Identified

  • DNS — Resolves domain names to IP addresses.
  • TCP — Reliable transport protocol (three-way handshake observed).
  • HTTP / HTTPS (TLS) — Web traffic; TLS handshakes for secure connections.
  • ICMP — Ping requests and replies for connectivity checks.
  • UDP — Background and connectionless traffic.
  • ARP, ICMPv6, IGMP — Local network and multicast management traffic.

Observations

  • DNS queries precede web connections for name resolution.
  • TCP handshakes occur before HTTP/HTTPS data transfer.
  • ICMP pings confirm network reachability and show round-trip times.
  • Background UDP traffic and repeated DNS queries likely come from OS or apps.
  • TLS encryption hides HTTP payload contents from packet inspection.
